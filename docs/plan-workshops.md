# Workshops Admin UI — Implementation Plan

## Overview

Add a Workshops section to the admin app for managing webinars (the Rails model is `Webinar`, the admin routes are `/admin/webinars`). The UI will be at `/workshops/*`.

---

## API Reference (Rails backend)

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/admin/webinars` | Paginated list (params: `page`, `per`) |
| GET | `/admin/webinars/:uuid` | Single webinar |
| POST | `/admin/webinars` | Create webinar |
| PATCH | `/admin/webinars/:uuid` | Update webinar |
| GET | `/admin/webinars/:uuid/invitees` | List invitees |
| POST | `/admin/webinars/:uuid/invitees` | Create single invitee (`email`) |
| POST | `/admin/webinars/:uuid/invitees/batch` | Batch-create invitees (`emails` newline/space-separated) |
| DELETE | `/admin/webinars/:uuid/invitees/:invitee_uuid` | Remove invitee |
| GET | `/admin/webinars/:uuid/companies` | List attendee companies |
| POST | `/admin/webinars/:uuid/start` | Mark webinar as started |
| POST | `/admin/webinars/:uuid/send_welcome_emails` | Send welcome emails |
| POST | `/admin/webinars/:uuid/send_reminder_emails` | Send reminder emails |
| POST | `/admin/webinars/:uuid/send_followup_emails` | Send followup emails |

### Permitted fields (create/update)

```
title, scheduled_at, streaming_url, referrer_id,
welcome_email_body, followup_email_body, reminder_email_body
```

Note: `invite_code` is auto-generated by the backend on create. The UI may leave it out entirely.

### Response shapes

**Webinar (admin)**

```json
  "title": "...",
  "scheduled_at": "2026-05-15T14:00:00Z",
  "streaming_url": "...",
  "invite_code": "...",
  "welcome_email_body": "...",
  "followup_email_body": "...",
  "reminder_email_body": "...",
  "started_at": null,
  "referrer": null,
  "invitees_count": 12,
  "attendees_count": 3
}
```

**Invitee**
```json

  "uuid": "...",
  "email": "user@example.com",
  "invite_code": "...",
  "company_created": false,
  "welcome_email_sent_at": null,
  "reminder_email_sent_at": null

```

**Attendee company**
```json
{
  "company_slug": "...",
  "company_name": "...",
  "user_email": "...",
  "followup_email_sent_at": null
}
```

## Component Breakdown

### `lib/workshops/api.ts`

Types: `Workshop`, `WorkshopInvitee`, `WorkshopCompany`.

Functions:
- `fetchWorkshops(page, per)` → `{ workshops, pagination }`
- `fetchWorkshop(uuid)` → `Workshop`
- `createWorkshop(data)` → `Workshop`
- `updateWorkshop(uuid, data)` → `Workshop`
- `fetchInvitees(uuid)` → `WorkshopInvitee[]`
- `batchAddInvitees(uuid, emails)` → `WorkshopInvitee[]`
- `removeInvitee(uuid, inviteeUuid)` → `void`
- `fetchCompanies(uuid)` → `WorkshopCompany[]`
- `startWorkshop(uuid)` → `void`
- `sendWelcomeEmails(uuid)` → `void`
- `sendReminderEmails(uuid)` → `void`
- `sendFollowupEmails(uuid)` → `void`

---

### `app/(dashboard)/workshops/layout.tsx`

Minimal `SidebarInset` wrapper identical to `downloads/layout.tsx`. ~14 lines.

---

### `app/(dashboard)/workshops/page.tsx`

Client component. Loads workshops on mount, shows `<WorkshopsTable />`. Has a "New Workshop" button in `<PageHeader />`. ~30 lines.

---

### `app/(dashboard)/workshops/new/page.tsx`

Client component. Renders `<WorkshopForm />` in "create" mode. On success, navigates to `/workshops/[uuid]`. ~15 lines.

---

### `app/(dashboard)/workshops/[uuid]/page.tsx`

Client component. Reads `uuid` from params. Loads workshop + invitees + companies in parallel. Renders:
- `<WorkshopForm />` in "edit" mode
- `<WorkshopActions />` (start/email buttons)
- `<InviteesPanel />`
- `<CompaniesPanel />`
~40 lines.

---

### `components/workshops/workshops-table.tsx`

Client-side paginated table (same pattern as `OrganizationsTable`). Columns: Title, Date, Invite Code, Invitees, Attendees. Clicking a row navigates to the show page. ~80 lines.

---

### `components/workshops/workshop-form.tsx`

Shared create/edit form using shadcn `Input`, `Label`, `Textarea`, `Button`. Fields:
- Title (required)
- Scheduled at — datetime-local input
- Streaming URL
- Welcome / Reminder / Followup email body (textareas)

Accepts `initial?: Workshop` and `onSave(data)` callback. Handles its own save state ("saving" / "saved" / error). ~95 lines.

---

### `components/workshops/invitees-panel.tsx`

- Heading + count badge
- Textarea for pasting emails (one per line or space-separated)
- "Add Invitees" button → calls `batchAddInvitees`, refreshes list
- Renders `<InviteeRow />` for each invitee
~60 lines.

---

### `components/workshops/invitee-row.tsx`

Table row showing: email, invite code, status badges (welcome sent, reminder sent, signed up). Trash icon button to remove. ~35 lines.

---

### `components/workshops/companies-panel.tsx`

Read-only table of attendee companies. Columns: Company, User Email, Followup Sent. ~40 lines.

---

### `components/workshops/workshop-actions.tsx`

Card with action buttons: Start Workshop, Send Welcome Emails, Send Reminder Emails, Send Followup Emails. Each shows a loading spinner and uses `toast` for feedback. Disabled appropriately (e.g. Start disabled if already started). ~65 lines.

## Design Decisions

- **Identifier:** Use `uuid` throughout (the Rails model uses `uuid` as the URL param via `to_param`)
- **Invite code:** Auto-generated by backend; shown read-only on the show page, not editable
- **Batch invitees:** A textarea where you paste emails (newline or space separated) is simplest and matches the `create_batch` endpoint that splits on whitespace
- **No delete:** The Rails controller has no destroy action for webinars, so no delete UI
- **Email bodies:** Plain textareas (the backend renders them as markdown, so users write markdown)
- **Pagination:** The index uses server-side pagination (`page`/`per` params) since the list could grow large
